Overview

Where this fits

The lecture moves from probability theory into estimation. In probability, we start by assuming we already know the distribution and the parameters. However, when dealing with real data, we rarely know this and instead we have to rely on the observed data to estimate them.

Estimation: use observed data to learn unknown quantities.
Maximum likelihood: choose the parameter values under which the observed data would be most likely.
Later topics: estimator properties, distribution theory, hypothesis testing, and bootstrap.
One-sentence goal
Data are clues. MLE asks: “Which parameter makes these clues most believable?”
Estimation

Estimation

Imagine you see a sequence of daily returns, trade counts, or default indicators. You do not directly see the hidden mechanism (i.e. the data generating process) that generated them. An estimator is a rule that converts the data into a guess about that hidden mechanism.

Example: If transaction counts look like Poisson data, the unknown parameter might be λ, which is the average number of transactions in a fixed time window.

Start with data

MLE starts with data

A clearer way to introduce MLE is: we observe data first, then we choose a plausible probability model, and then we use MLE to estimate the model’s unknown parameters.

The logic in one glance

Step 1. Suppose yi is the income of person i. So the sample is y₁, y₂, …, yₙ.
Step 2. Add axes so the observations become values on an income scale.
Step 3. Group the observed values into a histogram to see the sample’s shape.
Step 4. If the histogram looks roughly bell-shaped, we might assume a Normal distribution.
Step 5. MLE then estimates the unknown parameters of that Normal distribution.
Assumption
Yi ~ N(μ, σ²)

MLE chooses
(μ̂, σ̂²) = argmaxμ,σ² L(μ, σ²; Y)

In plain English: MLE picks the values of the centre and the spread that make the observed incomes look most plausible under the assumed Normal model.

Notation and Simplifications

Notation and Simplifications

We observe data points Y₁, …, Yₙ. However, we do not know the paramters vector θ. The likelihood function is the joint density of the observed data, viewed as a function of θ:

L(θ; Y) = f(Y; θ)

The maximum likelihood estimate is:

θ̂ = argmaxθ L(θ; Y)

Maximum Likelihood estimator θ̂ is the value of θ that maximizes the likelihood of observing the data in hand.

It’s easier with logs

In practice we usually maximize the log-likelihood:

ℓ(θ; y) = ln L(θ; y)

Because the log function is monotonically increasing transformation, it reaches its maximum at the same parameter value as the likelihood. Logs also turn products into sums, which is much easier for algebra and software.

Useful memory hook: Product of many tiny probabilities can be difficult to deal with. Sum of logs is friendlier.

IID data: the big simplification

If observations are independent and identically distributed, the joint likelihood is the product of individual observations densities:

f(y; θ) = ∏i=1n f(yi; θ)

Taking logs gives:

ℓ(θ; y) = Σi=1n ln f(yi; θ)

Sometimes the ML estimator is available in closed form. When not, MLE can still be found by numerical tools (e.g. Newton-Raphson iteration).

If a random variable X follows a Poisson distribution with parameter λ (where λ is the average number of transactions in five-minutes windows), then the probability of observing a specific count x is:

f(x; λ) = P(X = x) = e−λ λx / x!

Assuming the counts are independent and come from the same Poisson distribution, the likelihood is the product of the marginal densities.

Taking logs of the joint density (i.e. logs of the product of the densities of the 12 independent observations), gives the log-likelihood:

ℓ(λ; Y) = Σi=1n [−λ + Yi log(λ) − log(Yi!)]

Conditional MLE

Conditional versus unconditional MLE

An unconditional model uses the same marginal density for every observation. A conditional model allows the distribution of Y to change conditional on another variable(s) X.

Unconditional model: suppose income is drawn from one Normal distribution for the whole population.

Yi ~ N(μ, σ²)

This says every person’s income is described using the same centre μ and the same spread σ². Education is not used in the model.

Conditional model: suppose income depends on education. Let Xᵢ be the years of education of person i. We can assume income is Normally distributed conditional on education:

Yi | Xi ~ N(μi, σ²)
μi = β₀ + β₁xi

Now each person can have a different Normal distribution because each person can have a different education level. The model still has only a small number of parameters in this case: β₀, β₁, and σ² that are found by maximizing the conditional log-liklihood.

Interpretation: in the conditional model, the Normal income distribution can shift depending on education. A person with more education may have a distribution centred further to the right. The curve changes with x, but the rule linking education to the centre of the distribution is the same for all observations: μi = β₀ + β₁xi.

Know more: Poisson distribution

The Poisson distribution is used when we count how many times events occur in a fixed interval of time or space, assuming we know the average rate of events occurrence across the fixed intervals.

Simple example

If a café gets 3 customers on average every 10 minutes, we can use a Poisson model to ask: how many customers might arrive in the next 10 minutes? Here, the key parameter is λ = 3, the average number of events in the interval.

When it is useful

  • the variable is a count: 0, 1, 2, 3, …
  • events are counted over a fixed period or area;
  • events are roughly independent;
  • the average rate is reasonably stable;
  • events are not heavily clustered because of the same cause.

Financial examples

Poisson models are often useful for rare or countable events: number of trades per minute, corporate defaults per year, insurance claims per month, price jumps of an asset in a day, etc.

Memory hook: Poisson counts events over time or space. The exponential distribution is anther distribution that is related and useful, but it is used for finding the waiting time between events (i.e. how long do we need to wait until the next Poisson event).

Know more: Normal distribution

The Normal distribution is popular because it gives a simple model for variables that cluster around an average, with fewer observations far away from the centre.

Why it is often used

  • many small random effects can add up to something approximately Normal;
  • it is mathematically convenient;
  • it is easy to interpret using a mean and a variance;
  • it often provides a reasonable first approximation.

Financial caution

Financial returns are often not perfectly Normal. They can have fat tails, skewness, volatility clustering, and jumps. So the Normal distribution is a useful starting point, but not always the final answer.

Simple intuition: use a Normal model when many small forces combine and most observations are expected to be near the average. Be more careful when rare, large, asymmetric events matter.

Why Poisson for counts?

The sample consists of non-negative integer counts: 0, 1, 2, 3, and so on. A Normal distribution would allow negative values, which makes less sense for a count variable. The Poisson distribution is often a natural first model for counts observed over equal time windows.

Why use a Normal distribution here?

The Normal assumption is not claiming income is perfectly Normal. It is a simple way to model a variable using a centre and a spread. In the conditional model, the centre changes with education: μᵢ = β₀ + β₁xᵢ.

The important MLE idea is not the Normal curve itself. The important idea is that once we specify a conditional density for Y given X, we can build and maximize the conditional likelihood.
