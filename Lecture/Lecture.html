Overview

Where this fits

The lecture moves from probability theory into estimation. In probability, we often start by pretending we already know the distribution and its parameters. In real data work, we usually do not know them, so we use data to estimate them.

Estimation: use observed data to learn unknown quantities.
Maximum likelihood: choose the parameter values under which the observed data would be most likely.
Later topics: estimator properties, distribution theory, hypothesis testing, and bootstrap.

One-sentence goal

Data are clues. MLE asks: “Which parameter makes these clues most believable?”

Estimation

Estimation in plain English

Imagine you see a sequence of daily returns, trade counts, or default indicators. You do not directly see the hidden mechanism that generated them. An estimator is a rule that converts the data into a guess about that hidden mechanism.

Example: If transaction counts look like Poisson data, the unknown parameter might be λ, the average number of transactions in a fixed time window.

Data to model

MLE starts with data

A clearer way to introduce MLE is this: we observe data first, then we choose a plausible probability model, and then we use MLE to estimate the model’s unknown parameters.

Easy explanation

The logic in one glance

Step 1. Suppose yi is the income of person i. So the sample is y1, y2, …, yn.
Step 2. Add axes so the observations become values on an income scale.
Step 3. Group the observed values into a histogram to see the sample’s shape.
Step 4. If the histogram looks roughly bell-shaped, we might assume a Normal distribution.
Step 5. MLE then estimates the unknown parameters of that Normal distribution.
Assumption
Yi ~ N(μ, σ²)
MLE chooses
(μ̂, σ̂²) = argmaxμ,σ² L(μ, σ²; Y)

In plain English: MLE picks the values of the centre and the spread that make the observed incomes look most plausible under the assumed Normal model.

Estimated mean μ̂
Estimated s.d. σ̂
This is the core of MLE: data + distributional assumption + parameter values chosen to fit the data as well as possible.
Model assumption

The setup

We assume a probability model, such as Normal(μ, σ²) or Poisson(λ), but we do not know the parameter.

The question

For each possible parameter value, ask: how compatible is this value with the data I actually saw?

The choice

The MLE is the parameter value that gives the highest likelihood.

Likelihood is not “the probability that the parameter is true.” It is a score for how well a parameter explains the observed data.

Core notation

The core notation

We observe data points Y₁, …, Yₙ. The model has an unknown parameter θ. The likelihood function is the joint density of the observed data, viewed as a function of θ:

L(θ; Y) = f(Y; θ)

The maximum likelihood estimate is:

θ̂ = argmaxθ L(θ; Y)

Why logs?

In practice we usually maximize the log-likelihood:

ℓ(θ; y) = ln L(θ; y)

Because the log function is increasing, it reaches its maximum at the same parameter value as the likelihood. Logs also turn products into sums, which is much easier for algebra and software.

Useful memory hook: Product of many tiny probabilities can be awkward. Sum of logs is friendlier.

IID data: the big simplification

If observations are independent and identically distributed, the joint likelihood is the product of individual densities:

f(y; θ) = ∏i=1n f(yᵢ; θ)

Taking logs gives:

ℓ(θ; y) = Σi=1n ln f(yᵢ; θ)

Poisson example

Connecting the log-likelihood to the data

Suppose we count the number of transactions in consecutive five-minute windows for a hypothetical asset traded on exchange XYZ. The observed counts are:

132423052143

So here Y₁ = 1, Y₂ = 3, and so on. There are n = 12 observations and their total is ΣYᵢ = 30.

If a random variable X follows a Poisson distribution with parameter λ, then the probability of observing a count x is:

f(x; λ) = P(X = x) = e−λ λx / x!

For the first few observations in our sample, this means:

Observed countProbability contribution
Y₁ = 1f(1; λ) = e−λλ1/1!
Y₂ = 3f(3; λ) = e−λλ3/3!
Y₃ = 2f(2; λ) = e−λλ2/2!

Assuming the counts are independent and come from the same Poisson distribution, the likelihood multiplies all these probability contributions:

L(λ; Y) = f(1;λ)f(3;λ)f(2;λ)f(4;λ)f(2;λ)f(3;λ)f(0;λ)f(5;λ)f(2;λ)f(1;λ)f(4;λ)f(3;λ)
Taking logs turns the product into a sum.

The general log-likelihood is:

ℓ(λ; Y) = Σi=1n [−λ + Yᵢ log(λ) − log(Yᵢ!)]

For this particular sample, substitute n = 12 and ΣYᵢ = 30:

ℓ(λ; Y) = −12λ + 30 log(λ) − Σi=112 log(Yᵢ!)
Interpretation: the first part, −12λ, reflects the twelve observation windows. The second part, 30 log(λ), reflects the total number of observed transactions. The final factorial term is determined by the data and does not change as λ changes. Therefore, the graph below mainly shows how plausible different values of λ are for this exact sample.

Conditional models

Conditional versus unconditional MLE

An unconditional model uses one distribution for everyone. A conditional model allows the distribution of Y to change when we know another variable X.

Unconditional model: suppose income is drawn from one Normal distribution for the whole population.

Yᵢ ~ N(μ, σ²)

This says every person’s income is described using the same centre μ and the same spread σ². Education is not used in the model.

Conditional model: suppose income depends on education. Let Xᵢ be the years of education of person i. We can assume income is Normally distributed conditional on education:

Yᵢ | Xᵢ = xᵢ ~ N(μᵢ, σ²)
μᵢ = β₀ + β₁xᵢ

Now each person can have a different Normal distribution because each person can have a different education level. The model still has only a small number of parameters: β₀, β₁, and σ².

Distribution help for this example
Interpretation: in the conditional model, a person with more education may have a Normal income distribution centred further to the right. The curve changes with x, but the rule connecting education to the centre of the distribution is still simple.

Know more: Poisson distribution

The Poisson distribution is used when we count how many times something happens in a fixed interval of time or space, assuming we know the average rate.

Simple example

If a café gets 3 customers every 10 minutes on average, we can use a Poisson model to ask: how many customers might arrive in the next 10 minutes? Here, the key parameter is λ = 3, the average number of events in the interval.

When it is useful

  • the variable is a count: 0, 1, 2, 3, …
  • events are counted over a fixed period or area;
  • events are roughly independent;
  • the average rate is reasonably stable;
  • events are not heavily clustered because of the same cause.

Financial examples

Poisson models are often useful for rare or countable events: number of trades per minute, defaults per year, insurance claims per month, operational failures, price jumps, or bank runs/default events.

Memory hook: Poisson counts events over time or space. The exponential distribution is related, but it is used for waiting times between events.

Know more: Normal distribution

The Normal distribution is popular because it gives a simple model for variables that cluster around an average, with fewer observations far away from the centre.

Why it is often used

  • many small random effects can add up to something approximately Normal;
  • it is mathematically convenient;
  • it is easy to interpret using a mean and a variance;
  • it often provides a reasonable first approximation.

Financial caution

Financial returns are often not perfectly Normal. They can have fat tails, skewness, volatility clustering, and jumps. So the Normal distribution is a useful starting point, but not always the final answer.

Simple intuition: use a Normal model when many small forces combine and most observations are expected to be near the average. Be more careful when rare, large, asymmetric events matter.
