From lecture idea to coding exercise

In the lecture, MLE was introduced as a method for choosing parameter values that make the observed data most likely under an assumed distribution. Here, the data are daily S&P 500 log returns and the assumed distribution is Normal.

Important: this is a first applied MLE exercise, not a claim that market returns are perfectly Normal. Students should estimate the model first, then criticise it.

Data preparation

What are we modelling?

The raw data are daily closing values of the S&P 500 in 2017. We do not model the price level directly. Instead, we convert prices into daily returns (i.e. percentage growth rate):

rt = 100 × [log(Pt) − log(Pt−1)]

This produces a return series measured in percent. Returns are more suitable for this first model because they fluctuate around a relatively stable centre, unlike prices which trend over time (i.e. prices follow a unit root process as evident by their upward drift, while returns seem to be stationary).

What the code is doing

The code is split into small steps. Each step has a coding purpose and an econometric purpose. You should run the chunks in order.

What this code does: it reads the S&P 500 CSV file. If the local CSV is missing, it attempts to download the same 2017 series from FRED. It also standardises the date-column name.

What this code does: it turns price levels into daily log returns. The first price has no previous price, so there is one fewer return than price observation.
returns <- 100 * diff(log(sp500_clean$SP500))

What this code does: it calculates the closed-form MLEs for the Normal model. The MLE standard deviation divides by n, not n−1.
mu_hat <- mean(returns)
sigma_hat <- sqrt(mean((returns - mu_hat)^2))
annualised_volatility <- sigma_hat * sqrt(252)
Calculates the closed-form MLEs.
What this code does: it writes a negative log-likelihood function and asks optim() to minimise it. This gives the same MLE logic in a form that generalises to harder models.
negative_log_likelihood <- function(par, data) {
  mu <- par[1]
  sigma <- par[2]

  if (sigma <= 0) return(1e10)

  log_likelihood <- sum(dnorm(data, mean = mu, sd = sigma, log = TRUE))
  return(-log_likelihood)
}

mle_optim <- optim(
  par = c(mu = 0, sigma = 1),
  fn = negative_log_likelihood,
  data = returns
)

mle_optim$par
Shows the numerical optimiser result.

hist(returns, breaks = 25, probability = TRUE,

x_grid <- seq(min(returns), max(returns), length.out = 300)
lines(x_grid, dnorm(x_grid, mean = mu_hat, sd = sigma_hat), lwd = 2)

Likelihood in this seminar

The Normal likelihood for returns

We assume daily returns are independent draws from a Normal distribution:

rt ∼ N(μ, σ²)

For one return, the Normal density is:

f(rt | μ, σ) = 1/(σ√(2π)) exp{ −½ ((rt−μ)/σ)² }

The log-likelihood adds the log-density for every observed return:

ℓ(μ, σ) = Σt=1T log f(rt | μ, σ)
Plain English: MLE searches for the centre μ and spread σ that make this particular set of daily returns look most plausible under the Normal model.

Interactive model check

Histogram and fitted Normal curve

Move the slider to change σ. A very small σ makes the model too confident; a very large σ spreads probability too widely. The MLE balances these forces for the observed returns.

Checkpoint: If σ is too small, what happens to the likelihood of large observed returns?
Large returns become very unlikely. The model is too narrow, so actual observations far from μ receive very low density. The likelihood punishes this.
Checkpoint: Are μ̂ and σ̂ saying what will happen tomorrow?
No. They are estimates that make the observed 2017 sample most likely under the assumed Normal model. Forecasting is a separate step.
